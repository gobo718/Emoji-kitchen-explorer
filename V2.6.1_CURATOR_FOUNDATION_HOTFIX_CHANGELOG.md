# Billy Labs v2.6.1 — Curator Foundation Hotfix

## Purpose
Correct the public Curator visibility regression in v2.6.0 and make the new Museum Archive unmistakably accessible from the established Curator workflow.

## Changes
1. **Public Curator visibility**
   - The production GitHub Pages hostname is no longer treated as a development host.
   - Normal public visitors therefore do not receive the floating Curator launcher.
   - Deliberate `?curator=1` activation, remembered curator mode, localhost, and clearly named development/staging hosts remain supported.

2. **Museum Archive access**
   - Added a prominent **Open Museum Archive** action to the Curator header.
   - Added **Museum Archive** to the Curator navigation row.

3. **Release metadata**
   - Updated visible site labels, build metadata, release manifest, README, changelog, and Curator backup metadata to v2.6.1.

4. **Roadmap**
   - Marked v2.6.0 complete and v2.6.1 current.
   - Confirmed v2.7.0 Laboratory as the next major release.
