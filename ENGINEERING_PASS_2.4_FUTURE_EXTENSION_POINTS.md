# Pass 2.4 Future Extension Points

- A server-side or generated static index can implement the same canonical-ID contract without changing consumers.
- Laboratory pair validation should use `BillyMashups.has` or `resolve`.
- Collection import validation can use `resolveId` to reject orphaned member IDs.
- Curator tools can display an explicit “unresolved authored reference” state instead of substituting imagery.
- A future lazy-loaded dataset may provide a resolver adapter while preserving the exact/null contract.
